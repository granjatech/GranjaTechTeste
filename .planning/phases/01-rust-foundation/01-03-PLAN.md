**services/auth_service.rs** -- Implement by reading AuthService.cs carefully. Key methods:

1. `login(pool, dto, config) -> Result<LoginResponseDto, AppError>` -- Query `"Usuarios"` JOIN `"Perfis"` by email, verify BCrypt hash with `bcrypt::verify(&dto.senha, &usuario.senha_hash)`, generate JWT with `gerar_jwt()`. Return `LoginResponseDto { token }`.

2. `registrar(pool, dto, config) -> Result<UserDto, AppError>` -- Check duplicate email. Generate sequential code: query `SELECT MAX("Codigo") FROM "Usuarios"`, parse numeric part, increment (format "USR001", "USR002"...). Hash password with `bcrypt::hash(&dto.senha, 10)` (cost factor 10 matches .NET default). INSERT into `"Usuarios"`. If perfil_id == 3 (Financeiro) and produtores_ids provided, INSERT into `"FinanceiroProdutor"`.

3. `get_all(pool) -> Result<Vec<UserDto>, AppError>` -- Admin-only. SELECT with JOIN on Perfis.

4. `get_by_id(pool, id) -> Result<UserDetailDto, AppError>` -- Admin-only. SELECT user + JOIN Perfis + query FinanceiroProdutor for produtores_ids.

5. `update(pool, id, dto) -> Result<(), AppError>` -- Admin-only. Check email uniqueness (excluding current user). Update Usuarios. If senha provided, hash it. Manage FinanceiroProdutor: DELETE existing, INSERT new.

6. `delete(pool, id) -> Result<(), AppError>` -- Admin-only. Check no Granjas depend on user. DELETE FinanceiroProdutor associations. DELETE user.

7. `gerar_jwt(usuario_id, email, perfil_nome, config) -> Result<String, AppError>` -- Private helper. Per D-05: Claims with `nameid` (user ID as string), `email`, `role` (profile name), `exp` (8 hours), `iss` (config.jwt_issuer), `aud` (config.jwt_audience). Use `jsonwebtoken::encode` with `Header::new(Algorithm::HS256)` and `EncodingKey::from_secret(config.jwt_key.as_bytes())`.

**CRITICAL SQL notes:**
- ALL table/column names MUST be double-quoted PascalCase: `"Usuarios"`, `"SenhaHash"`, `"PerfilId"`
- Use `r#"..."#` raw strings for SQL with embedded quotes
- Use column aliases to map PascalCase DB columns to snake_case struct fields: `SELECT "Id" as id, "Nome" as nome` OR rely on `#[sqlx(rename)]` on the struct (check what Plan 02 used)
- Per D-02: Runtime queries only, no `sqlx::query!()` macro

**services/granja_service.rs** -- Implement by reading GranjaService.cs. Key methods:

```rust
#[utoipa::path(
    post, path = "/api/auth/login",
    request_body = LoginDto,
    responses((status = 200, body = LoginResponseDto))
)]
pub async fn login(
    pool: web::Data<PgPool>,
    config: web::Data<Config>,
    body: web::Json<LoginDto>,

pub async fn get_usuarios(
    pool: web::Data<PgPool>,
    claims: Claims,  // JWT extractor -- auth required
) -> Result<HttpResponse, AppError> {
    if claims.role != "Administrador" {
        return Err(AppError::Forbidden("Acesso restrito a administradores".into()));
```

```rust
use actix_web::web;

pub mod auth;

pub fn configure_routes(cfg: &mut web::ServiceConfig) {
    cfg.service(

                web::scope("/auth")
                    .route("/login", web::post().to(auth::login))
```

```rust
use actix_cors::Cors;
use tracing_actix_web::TracingLogger;
use utoipa::OpenApi;
use utoipa_swagger_ui::SwaggerUi;

#[derive(OpenApi)]
#[openapi(
    paths(
        handlers::auth::login,
        handlers::auth::registrar,
        handlers::auth::get_usuarios,
        handlers::auth::get_usuario,
        handlers::auth::update_usuario,

        handlers::granjas::get_granja,

    components(schemas(
        dto::auth::LoginDto,
        dto::auth::RegisterDto,
        dto::auth::LoginResponseDto,
        dto::auth::UserDto,
        dto::auth::UserDetailDto,
        dto::auth::UpdateUserDto,
        dto::granja::CreateGranjaDto,
        dto::granja::UpdateGranjaDto,
        // Add Granja model schema if needed
    )),
    tags(
        (name = "auth", description = "Autenticacao e gerenciamento de usuarios"),
        (name = "granjas", description = "CRUD de granjas")
    )
)]
struct ApiDoc;

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    // Initialize tracing
    tracing_subscriber::fmt()
        .with_env_filter(
            tracing_subscriber::EnvFilter::try_from_default_env()
                .unwrap_or_else(|_| tracing_subscriber::EnvFilter::new("info"))
        )
        .init();

    tracing::info!("Carregando configuracao...");
    let config = config::Config::from_env();

    tracing::info!("Conectando ao banco de dados...");
    let pool = db::create_pool(&config.database_url).await;

    let swagger_enabled = config.swagger_enabled;
    let allowed_origins = config.allowed_origins.clone();

    tracing::info!("Servidor iniciando na porta 8080...");

    let config_data = web::Data::new(config);
    let pool_data = web::Data::new(pool);

    HttpServer::new(move || {
        // CORS configuration (FOUND-02) -- explicit origins from config, not wildcard
        let mut cors = Cors::default()
            .allowed_methods(vec!["GET", "POST", "PUT", "DELETE", "OPTIONS"])
            .allowed_headers(vec![
                actix_web::http::header::AUTHORIZATION,
                actix_web::http::header::CONTENT_TYPE,
                actix_web::http::header::ACCEPT,
            ])
            .max_age(3600);

        for origin in &allowed_origins {
            cors = cors.allowed_origin(origin);
        }

        let mut app = App::new()
            .wrap(TracingLogger::default())
            .wrap(cors)
            .app_data(config_data.clone())
            .app_data(pool_data.clone())
            .configure(handlers::configure_routes)
            .route("/health", web::get().to(|| async {
                HttpResponse::Ok().json(serde_json::json!({"status": "ok"}))
            }));

        // Swagger UI (FOUND-04) -- conditionally enabled
        if swagger_enabled {
            app = app.service(
                SwaggerUi::new("/swagger-ui/{_:.*}")
                    .url("/api-docs/openapi.json", ApiDoc::openapi())
            );
        }

        app
    })
    .bind("0.0.0.0:8080")?
    .run()
    .await
}
```

**CORS details (FOUND-02):**
- Allowed origins read from `Config.allowed_origins` (loaded from `ALLOWED_ORIGINS` env var)
- Allowed methods: GET, POST, PUT, DELETE, OPTIONS
- Allowed headers: Authorization, Content-Type, Accept
- Max age: 3600 seconds
- NOT using `.allow_any_origin()` -- explicit origins only (security requirement T-1-05)

**Swagger details (FOUND-04):**
- UI served at `/swagger-ui/`
- OpenAPI JSON at `/api-docs/openapi.json`
- Conditionally enabled via `SWAGGER_ENABLED` env var
- All handlers annotated with `#[utoipa::path(...)]`
- All DTOs derive `ToSchema`

**Note:** The utoipa `swagger_enabled` check cannot be done inside the `#[derive(OpenApi)]` -- the conditional is on registering the SwaggerUi service. The `ApiDoc` struct is always compiled, but the `/swagger-ui/` route is only registered when `swagger_enabled` is true.

The `let swagger_enabled` and `let allowed_origins` must be cloned/copied before the `move ||` closure since Config is moved into `web::Data`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> /api/auth/login | Untrusted email+password from client |
| Client -> /api/auth/registrar | Untrusted registration data from client |
| Client -> /api/granjas/* | Authenticated requests with role-based access |
| JWT token -> Claims | Token from Authorization header validated server-side |
| Service -> PostgreSQL | Parameterized queries prevent SQL injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Information Disclosure | auth_service.rs JWT | mitigate | JWT secret from Config (env var JWT_KEY), never hardcoded. EncodingKey::from_secret uses runtime value. |
| T-1-02 | Tampering | auth_service.rs BCrypt | mitigate | bcrypt crate handles constant-time comparison internally. Cost factor 10 (matches .NET). bcrypt::hash and bcrypt::verify from vetted crate, not hand-rolled. |
| T-1-03 | Elevation of Privilege | handlers/auth.rs, handlers/granjas.rs | mitigate | Admin endpoints check `claims.role != "Administrador"` before proceeding. Granjas CUD blocks Financeiro role. Service-layer role filtering for list operations (Admin=all, Produtor=own, Financeiro=via junction table). |

| T-1-05 | Spoofing (CORS) | main.rs | mitigate | CORS configured with explicit `allowed_origin()` calls from Config, not `allow_any_origin()`. Origins loaded from ALLOWED_ORIGINS env var. |
