|----------|-------------|
| Browser -> Rust API | All user input crosses this boundary via API calls |
| localStorage -> Auth store | JWT token read from untrusted storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | UsuariosView admin check | mitigate | Client-side `isAdmin` computed check prevents UI access; backend `/auth/usuarios` endpoint enforces `Authorize(Roles="Administrador")` server-side |
| T-05-02 | Tampering | ProfileView password change | mitigate | Requires `senhaAtual` field verified server-side; Vuetify `:rules` validates matching passwords client-side |
| T-05-03 | Information Disclosure | AuditoriaView | accept | Audit logs visible to all authenticated users (same as .NET version); no PII beyond user names |
| T-05-04 | Elevation of Privilege | UsuariosView perfilId | mitigate | `v-select` constrains to valid role IDs (1,2,3); backend validates perfilId on register/update |
</threat_model>
