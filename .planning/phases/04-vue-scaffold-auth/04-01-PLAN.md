1. Create `granjatech-frontend/src/services/api.ts` (per D-09, D-10):
   ```typescript
   import axios from 'axios'

   const api = axios.create({
     baseURL: import.meta.env.VITE_API_URL || 'http://localhost:5099/api',
     timeout: 30000,
     headers: { 'Content-Type': 'application/json' },
   })

   // Request interceptor: attach Bearer token from localStorage
   api.interceptors.request.use((config) => {
     const token = localStorage.getItem('token')
     if (token) {
       config.headers.Authorization = `Bearer ${token}`
     }
     return config
   })

   // Response interceptor: redirect to /login on 401
   api.interceptors.response.use(
     (response) => response,
     (error) => {
       if (error.response?.status === 401) {
         localStorage.removeItem('token')
         window.location.href = '/login'
       }
       return Promise.reject(error)
     }
   )

   export default api
   ```

2. Create `granjatech-frontend/src/stores/auth.ts` (per D-11, D-12):
   - Define `JwtPayload` interface: `{ nameid: string; email: string; role: string; exp: number }`
   - Use `defineStore('auth', () => { ... })` (setup store syntax)
   - `token` ref initialized from `localStorage.getItem('token')`
   - `user` ref initialized by decoding token with `jwtDecode<JwtPayload>()` (from `jwt-decode`)
   - On hydration, check `exp` claim: if `exp * 1000 < Date.now()`, clear token (expired token protection per Research pitfall 4)
   - `isAuthenticated` computed: `!!token.value && !!user.value`
   - `login(email: string, senha: string)` async action: POST to `/auth/login` with `{ email, senha }`, extract `response.data.token`, decode, save to localStorage
   - `logout()` action: clear token, user, remove localStorage 'token'
   - Return: `{ token, user, isAuthenticated, login, logout }`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Rust API | User input (email/senha) crosses from untrusted browser to backend |
| localStorage -> App | Stored JWT token read on app init, could be tampered |
| Axios interceptor -> Backend | Bearer token attached to every request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | localStorage token storage | accept | JWT in localStorage is the established pattern (D-11), matching React. Backend enforces 8-hour expiry. 401 interceptor clears token on expiry. |
| T-04-02 | Tampering | XSS injecting malicious script | mitigate | Vue template auto-escaping prevents XSS. No v-html usage in any component. All user input goes through Vuetify form components. |
| T-04-03 | Spoofing | Expired/forged JWT token | mitigate | Auth store checks exp claim on hydration, rejects expired tokens. Backend validates JWT signature server-side on every request. |
| T-04-04 | Elevation of Privilege | Client-side role filtering bypass | accept | Navigation filtering is UI-only convenience. Backend enforces role-based access on every endpoint. Client cannot escalate privileges. |
| T-04-05 | Spoofing | Open redirect after login | mitigate | Router guard only redirects to known routes defined in router config. No dynamic redirect URL parameter accepted. |
</threat_model>
