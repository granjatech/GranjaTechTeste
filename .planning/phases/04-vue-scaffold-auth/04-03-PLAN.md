<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete Vue 3 + Vuetify frontend with:
- Login page with gradient background and form validation
- JWT authentication against Rust backend (POST /api/auth/login)
- Navigation drawer with 13 role-filtered menu items
- App bar with dark mode toggle, font scale controls, user menu
- PageContainer with breadcrumbs on all routes
- 15 routes with placeholder views for Phase 5
  </what-built>
  <how-to-verify>
**Prerequisites:** Ensure the Rust backend is running at localhost:5099. Start the Vue dev server:
```bash
cd granjatech-frontend && npm run dev
```

**Test 1 - Route protection:**
1. Open http://localhost:5173 (or whatever port Vite shows)
2. Verify you are redirected to /login (no flash of dashboard content)
3. Expected: Login page with green gradient background, GranjaTech branding, email/senha fields

**Test 2 - Login flow:**
1. Enter valid credentials (an admin user from the existing database)
2. Click "Entrar"
3. Expected: Redirect to / (Dashboard), navigation drawer appears with all 13 menu items
4. Try invalid credentials: should show "Email ou senha invalidos." error alert

**Test 3 - Navigation:**
1. Click several menu items in the drawer (Granjas, Lotes, Financeiro, etc.)
2. Expected: Each shows placeholder page with "Em construcao" message
3. Verify breadcrumbs update correctly (e.g., Dashboard > Granjas)
4. On desktop (md+): drawer is permanent, always visible
5. Resize browser to mobile width: drawer collapses, hamburger menu appears in app bar

**Test 4 - Dark mode:**
1. Click the moon/sun icon in the app bar
2. Expected: Entire UI switches between light (#F8F9FA background) and dark (#121212 background) mode
3. Toggle back and forth
4. Refresh the page: dark mode preference should persist (from localStorage)

**Test 5 - Font scale:**
1. Click the font increase button (A+) in the app bar several times
2. Expected: All text grows larger proportionally
3. Click font decrease (A-) to go smaller
4. Verify buttons disable at min (0.85) and max (1.3) scale
5. Click reset button: should return to default size and light mode
6. Refresh: font scale should persist

**Test 6 - User menu:**
1. Click the user name/avatar in the app bar
2. Expected: Dropdown with "Perfil" and "Sair" options
3. Click "Perfil": navigate to /perfil page
4. Click "Sair": redirect to /login, drawer disappears

**Test 7 - Session persistence:**
1. Log in, then close and reopen the browser tab
2. Expected: Still authenticated, drawer visible, no redirect to login
  </how-to-verify>
  <resume-signal>Type "approved" if all 7 tests pass, or describe any issues found</resume-signal>
